🔍 RAG Fundamentals & Vector Databases

Semantic Search with Azure AI Search and Embeddings

What is RAG (Retrieval-Augmented Generation)?

STEP 1 📝 Query User question
STEP 2 🔢 Embed Convert to vector
STEP 3 🔍 Search Find similar docs
STEP 4 📄 Retrieve Get context
STEP 5 🤖 Generate LLM response

🔢 Tokenization & Embeddings

  • Tokenization: Split text into tokens (words/subwords)
  • Embeddings: Convert tokens to dense vectors
  • Dimensions: text-embedding-3-small = 1536 dims
  • Similarity: Cosine similarity for matching

🗄️ Vector Database Options

  • Azure AI Search: Native Azure, hybrid search
  • Azure Cosmos DB: Vector search + NoSQL
  • FAISS: Open-source, in-memory
  • Pinecone: Managed, serverless
💡
Why RAG for Banking? Keep knowledge current (policies, rates), provide source citations for compliance, and avoid hallucinations with grounded responses.

⚖️ RAG vs SLM vs MCP: Comprehensive Comparison

Choosing the Right Architecture for Your Banking Use Case

Aspect 🔍 RAG 🧠 SLM 🔌 MCP
What It IsRetrieve relevant docs → inject into LLM context → generate responseSmaller, specialized models (1-7B params) fine-tuned for specific tasksStandardized protocol for LLMs to connect to external tools/data sources
Best ForKnowledge-intensive Q&A, policy lookup, document searchHigh-volume, narrow tasks (classification, extraction, routing)Tool orchestration, real-time data access, multi-system integration
LatencyMedium (retrieval + generation)Low (small model, fast inference)Variable (depends on tool calls)
CostMedium (embedding + LLM + vector DB)Low (can run on-premise/edge)Medium (LLM + tool execution)
Azure ServicesAI Search + OpenAI + Blob StorageAzure ML + Container Apps + ONNX RuntimeAzure Functions + API Management + OpenAI

🏦 Banking Architecture Decision Matrix

Choose RAG When:

  • Policy/procedure Q&A
  • Product information lookup
  • Compliance document search
  • Knowledge changes frequently
  • Need citation/source tracking

Choose SLM When:

  • Intent classification (10K+ req/min)
  • Entity extraction (names, amounts)
  • Sentiment analysis
  • Data privacy requires on-premise
  • Latency < 100ms required

Choose MCP When:

  • Real-time account data needed
  • Multi-system orchestration
  • Transaction execution
  • Dynamic tool selection
  • Agent needs to "act" not just "answer"
💡
Hybrid Pattern: Most production banking agents combine all three: SLM for routing → RAG for knowledge → MCP for actions. Example: Classify intent (SLM) → Lookup policy (RAG) → Execute transfer (MCP).

⚖️ RAG Chat Flow: Trade-offs & Best Practices

Production Considerations for Banking Deployments

✅ Advantages

  • Pre-built template — deploy in hours, not weeks
  • Visual debugging — trace each node's output
  • Built-in evaluation — groundedness, relevance, coherence
  • Version control — track prompt changes
  • Managed deployment — auto-scaling endpoints
  • Azure-native — integrates with Key Vault, Monitor, AD

⚠️ Limitations

  • Azure lock-in — not portable to other clouds
  • Limited custom logic — complex flows need Python nodes
  • Cost at scale — managed endpoints can be expensive
  • Cold start — first request after idle can be slow
  • Region availability — not all regions support all features
  • Debugging complex flows — can be challenging at scale

🔍 Azure AI Search: Conversational Search

Enabling Multi-Turn Conversations on Indexed Banking Data

What is Conversational Search?

Conversational search combines Azure AI Search with Azure OpenAI to enable natural, multi-turn conversations over your indexed data. Unlike simple keyword search, it understands context, resolves references, and maintains conversation state.

🔄 Traditional Search

User: "wire transfer limit"

Result: List of documents containing those keywords

❌ No context, no follow-up, no understanding

💬 Conversational Search

User: "What's the wire transfer limit?"

AI: "For personal accounts, the daily limit is $50,000..."

User: "What about business accounts?"

AI: "Business accounts have a $250,000 daily limit..."

✅ Context-aware, multi-turn, cited sources

🎯 Semantic Ranking & Hybrid Search

Getting the Best Results from Azure AI Search

Search Modes Comparison

📝 Keyword Search (BM25)

  • Traditional text matching
  • Fast and cheap
  • Good for exact terms
  • Misses synonyms
Query: "wire transfer" → Finds "wire transfer" only

🔢 Vector Search

  • Semantic understanding
  • Finds similar meaning
  • Handles synonyms
  • Higher compute cost
Query: "wire transfer" → Also finds "bank remittance", "fund transfer"

🔀 Hybrid Search ⭐

  • Combines both approaches
  • Best accuracy overall
  • RRF (Reciprocal Rank Fusion)
  • Recommended for production
Query: "wire transfer" → Best of both worlds

Semantic Ranker: The Secret Weapon

Initial Results 50 candidates
Semantic Ranker Deep understanding
Re-Ranked Results Top 5 most relevant

🏦 Banking Example: Semantic Ranker in Action

Query: "Can I reverse a payment I made yesterday?"

Without Semantic Ranker: Returns docs about "payment processing" and "yesterday's transactions" (keyword match)

With Semantic Ranker: Returns "Transaction Dispute Policy" and "Payment Reversal Procedures" (understands intent)

✅
Production Recommendation: Always use Hybrid Search + Semantic Ranker for banking applications. The small additional cost ($1/1000 queries for semantic ranking) is worth the significant accuracy improvement.
